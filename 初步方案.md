## 三、核心模块详细设计（C++ 实现）

### 3.1 热区 `HotZone<K, V>`

**需求**：
- 存储键值对，并记录访问次数。
- 支持快速查找（哈希表）。
- 支持快速获取当前访问次数最小的键（用于淘汰）。
- 支持更新某个键的访问次数（频率增加后可能需要调整排序）。

**设计选择**：
- 主存储：`std::unordered_map<K, Entry>`，其中 `Entry` 包含值 `V`、频率 `size_t`，以及可选的在堆中的索引（如果实现索引堆）。
- 淘汰策略：由于热区容量 `H` 通常很小（如 128），我们可以**不实现索引堆**，而是每次需要淘汰时扫描整个热区找到频率最小的元素，复杂度 O(H)，对于常数 H 可接受。这样极大简化代码，也更适合教学。
- 频率更新：当命中热区时，增加频率，但无需调整顺序（只在淘汰时扫描）。这可能导致淘汰不够精确（可能淘汰的不是全局最小），但因为是教学项目，可以在文档中讨论更精确的实现。

**简化版实现**：

```cpp
template<typename K, typename V>
class HotZone {
private:
    struct Entry {
        V value;
        size_t freq;
    };
    std::unordered_map<K, Entry> data_;
    size_t capacity_;

public:
    explicit HotZone(size_t capacity) : capacity_(capacity) {}

    bool contains(const K& key) const { return data_.find(key) != data_.end(); }

    V& at(const K& key) { return data_.at(key).value; }

    void insert(const K& key, const V& value, size_t freq = 0) {
        data_[key] = {value, freq};
    }

    void increase_freq(const K& key) {
        auto it = data_.find(key);
        if (it != data_.end()) ++it->second.freq;
    }

    size_t size() const { return data_.size(); }

    // 获取频率最小的键（用于淘汰）
    std::pair<K, size_t> get_min_freq_key() const {
        if (data_.empty()) throw std::logic_error("HotZone empty");
        auto min_it = data_.begin();
        for (auto it = data_.begin(); it != data_.end(); ++it) {
            if (it->second.freq < min_it->second.freq) min_it = it;
        }
        return {min_it->first, min_it->second.freq};
    }

    void erase(const K& key) { data_.erase(key); }

    // 其他辅助方法...
};
```

**进阶版**：如果希望实现 O(log H) 淘汰，可以用 `std::priority_queue` 存储 `(freq, key)`，但更新频率时需要维护一致性，实现较复杂。教学上建议先使用扫描法，并在文档中介绍优化方向。

### 3.2 冷区 `ColdZone<K, V>`

**需求**：
- 存储所有非热区元素，支持 O(log n) 查找。
- 记录每个元素的访问次数，用于晋升决策。

**实现**：
- 使用 `std::map<K, std::pair<V, size_t>>`，键有序，查找 O(log n)。
- 提供 `increase_freq` 方法，并返回是否达到晋升阈值。

```cpp
template<typename K, typename V>
class ColdZone {
private:
    std::map<K, std::pair<V, size_t>> data_;  // key -> (value, freq)

public:
    bool contains(const K& key) const { return data_.find(key) != data_.end(); }

    V& at(const K& key) { return data_.at(key).first; }

    size_t get_freq(const K& key) const { return data_.at(key).second; }

    void insert(const K& key, const V& value, size_t freq = 0) {
        data_[key] = {value, freq};
    }

    void increase_freq(const K& key) {
        auto it = data_.find(key);
        if (it != data_.end()) ++it->second.second;
    }

    bool promote_if_needed(const K& key, size_t threshold) {
        auto it = data_.find(key);
        if (it == data_.end()) return false;
        if (it->second.second >= threshold) {
            // 返回true表示应该晋升
            return true;
        }
        return false;
    }

    void erase(const K& key) { data_.erase(key); }

    size_t size() const { return data_.size(); }
};
```

### 3.3 主类 `FreqPartitionDict<K, V>`

整合热区和冷区，提供统一接口。

**成员变量**：
- `HotZone<K, V> hot_zone_`
- `ColdZone<K, V> cold_zone_`
- `size_t hot_capacity_`
- `size_t promote_threshold_`  // 晋升阈值，如3次

**构造函数**：
```cpp
FreqPartitionDict(size_t hot_capacity = 128, size_t promote_threshold = 3)
    : hot_zone_(hot_capacity), hot_capacity_(hot_capacity), promote_threshold_(promote_threshold) {}
```

**查找 `get`**：

```cpp
bool get(const K& key, V& out) {
    // 1. 热区命中
    if (hot_zone_.contains(key)) {
        hot_zone_.increase_freq(key);
        out = hot_zone_.at(key);
        return true;
    }
    // 2. 冷区命中
    if (cold_zone_.contains(key)) {
        cold_zone_.increase_freq(key);
        out = cold_zone_.at(key);
        // 检查是否需要晋升
        if (cold_zone_.promote_if_needed(key, promote_threshold_)) {
            // 晋升
            promote(key);
        }
        return true;
    }
    return false;
}
```

**晋升 `promote`**：

```cpp
void promote(const K& key) {
    // 从冷区取出
    V value = cold_zone_.at(key);
    size_t freq = cold_zone_.get_freq(key);
    cold_zone_.erase(key);

    // 如果热区未满，直接插入
    if (hot_zone_.size() < hot_capacity_) {
        hot_zone_.insert(key, value, freq);
    } else {
        // 热区满，淘汰频率最小的键
        auto [min_key, min_freq] = hot_zone_.get_min_freq_key();
        V min_value = hot_zone_.at(min_key);
        hot_zone_.erase(min_key);
        // 放回冷区（保留频率）
        cold_zone_.insert(min_key, min_value, min_freq);
        // 插入新元素
        hot_zone_.insert(key, value, freq);
    }
}
```

**插入 `insert`**：

```cpp
void insert(const K& key, const V& value) {
    // 若已存在，更新值并重置频率（策略可选）
    if (hot_zone_.contains(key)) {
        hot_zone_.at(key) = value;
        // 频率保持不变？或者重置？这里我们保留频率
    } else if (cold_zone_.contains(key)) {
        cold_zone_.at(key) = value;
        // 频率保留
    } else {
        // 新元素：放入冷区，频率为0
        cold_zone_.insert(key, value, 0);
    }
}
```

**删除 `erase`**：

```cpp
bool erase(const K& key) {
    if (hot_zone_.contains(key)) {
        hot_zone_.erase(key);
        return true;
    } else if (cold_zone_.contains(key)) {
        cold_zone_.erase(key);
        return true;
    }
    return false;
}
```

**其他辅助方法**：`size()`, `clear()`, `contains()` 等。

### 3.4 关于内存与拷贝

- 值类型 `V` 应支持拷贝或移动语义。可以接受拷贝，但在性能敏感的场合可考虑使用 `std::unique_ptr` 或 `std::shared_ptr`，但教学项目建议保持简单。
- 键类型 `K` 需要同时满足哈希（用于 `unordered_map`）和比较（用于 `map`），因此最好同时提供 `std::hash<K>` 特化和 `operator<`。可以在文档中说明。

## 四、关键算法与复杂度分析

| 操作 | 热区命中 | 冷区命中（未晋升） | 冷区命中（晋升） |
|------|---------|------------------|----------------|
| 查找 | O(1) 哈希 + O(1) 频率更新 | O(log n) 树查找 + O(1) 频率更新 | 额外 O(H) 扫描淘汰（或 O(log H) 优化） |
| 插入 | O(1) 哈希插入 | O(log n) 树插入 | — |
| 删除 | O(1) 哈希删除 | O(log n) 树删除 | — |

**均摊分析**：假设访问模式为 Zipf 分布，高频元素数量约等于热区容量 H，则绝大多数访问命中热区，平均近似 O(1)。低频元素访问仍为 O(log n)。整体性能优于纯平衡树，且对热点变化有一定自适应能力。

- `std::unordered_map`
- `std::map`
- 本结构（不同热区容量）

测试场景：
1. **均匀随机访问**：所有键等概率，热区命中率低，预期本结构略慢于哈希表但快于树。

3. **热点迁移**：先访问一组热点（A组），然后突然切换至另一组（B组），测量查找延迟变化，观察自适应速度。
