# 设计文档

## 1. 设计目标

FreqPartitionDict 的设计目标是创建一个教学友好的数据结构，展示如何利用访问频率优化查找性能。主要目标包括：

1. **教学性**: 代码清晰易懂，适合学习数据结构和算法
2. **实用性**: 在倾斜访问模式下提供良好的性能
3. **可扩展性**: 易于修改和扩展不同的策略

## 2. 核心设计决策

### 2.1 双区架构

采用热区和冷区的分区策略基于以下观察：

- **访问倾斜性**: 大多数实际应用的访问模式遵循幂律分布（如 Zipf 分布），少数元素被频繁访问
- **局部性原理**: 程序倾向于重复访问最近使用过的数据

### 2.2 热区实现选择

热区使用 `std::unordered_map` 的理由：

- **O(1) 平均查找时间**: 哈希表提供常数时间查找
- **C++ 标准支持**: 无需额外依赖
- **教学价值**: 帮助理解哈希表的工作原理

**淘汰策略选择**：

采用扫描法（O(H)）而非堆（O(log H)）的原因：
1. 热区容量 H 通常较小（如 128），O(H) 扫描的开销可接受
2. 代码更简单，更适合教学
3. 在文档中说明了优化方向（可以使用索引堆优化到 O(log H)）

### 2.3 冷区实现选择

冷区使用 `std::map` 的理由：

- **O(log n) 查找时间**: 红黑树保证对数时间复杂度
- **有序性**: 支持范围查询（虽然本项目中未使用）
- **稳定性**: 最坏情况下仍保证 O(log n)

## 3. 晋升与淘汰策略

### 3.1 晋升条件

```
if (cold_zone_element.freq >= promote_threshold) {
    promote_to_hot_zone(element);
}
```

晋升阈值是可配置的参数：
- **阈值 = 1**: 第一次访问即晋升，适合热区容量大的场景
- **阈值 = 3-5**: 平衡策略，避免噪声访问导致频繁晋升
- **阈值 > 5**: 保守策略，只有真正高频的元素才进入热区

### 3.2 淘汰条件

当热区已满，新元素需要晋升时：

```
min_freq_element = find_min_frequency(hot_zone);
if (new_element.freq > min_freq_element.freq) {
    demote(min_freq_element);
    promote(new_element);
}
```

## 4. 替代设计考虑

### 4.1 使用伸展树 (Splay Tree)

**优点**：
- 自适应调整，频繁访问的元素靠近根
- 实现简单（相对于红黑树）

**缺点**：
- 最坏情况 O(n)
- 不适合需要稳定性能的场景

**结论**：本项目使用 `std::map` 更稳定，适合教学

### 4.2 使用 LFU 缓存

**区别**：
- LFU 需要全局维护所有元素的频率
- FreqPartitionDict 只在热区内维护精确频率

**适用场景**：
- LFU: 缓存大小固定，需要全局最优
- FreqPartitionDict: 数据量大，只需近似最优

## 5. 扩展方向

### 5.1 并发支持

可以通过以下方式添加并发支持：

1. **分区锁**: 热区和冷区使用独立的锁
2. **无锁热区**: 使用无锁哈希表实现热区
3. **RCU**: 读多写少场景使用 Read-Copy-Update

### 5.2 多级热区

可以扩展为多级热区：

```
L1 Hot Zone (极热) -> L2 Warm Zone (温热) -> Cold Zone
```

### 5.3 自适应阈值

根据访问模式动态调整晋升阈值：

```cpp
if (hot_hit_rate < target_rate) {
    promote_threshold--; // 更激进的晋升
} else if (promotions > threshold) {
    promote_threshold++; // 更保守的晋升
}
```

## 6. 教学要点

### 6.1 数据结构对比

| 特性 | 哈希表 | 平衡树 | FreqPartitionDict |
|-----|-------|-------|------------------|
| 查找 | O(1) 平均 | O(log n) | O(1) 热区 / O(log n) 冷区 |
| 插入 | O(1) 平均 | O(log n) | O(log n) |
| 删除 | O(1) 平均 | O(log n) | O(log n) |
| 有序性 | 否 | 是 | 否（热区）/ 是（冷区） |
| 适用场景 | 均匀访问 | 有序遍历 | 倾斜访问 |

### 6.2 关键学习点

1. **时间-空间权衡**: 热区用空间换时间
2. **启发式算法**: 频率预测未来访问的启发式
3. **均摊分析**: 为什么平均复杂度接近 O(1)
4. **工程实践**: 如何在清晰性和性能间取舍
